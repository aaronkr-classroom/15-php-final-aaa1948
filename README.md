# PHP 기말고사 실기 평가

## 개요

이번 실기 평가는 수업에서 학습한 PHP, MySQL, 데이터베이스 기반 웹사이트 개발 내용을 바탕으로 진행됩니다.

실기 평가는 **오픈북(Open Book)** 형태로 진행됩니다.

학생들은 제공된 CMS 프로젝트를 실행하고, 누락된 `article.php` 파일을 완성한 후 정상적으로 동작하도록 만들어야 합니다.

GitHub 저장소에는 다음 자료가 포함되어 있습니다.

* 전체 CMS 프로젝트
* 데이터베이스 파일


## Part 2. 개념 질문 (20점)

다음 질문에 간단히 답하시오.

### 1. prepare() 와 execute() 를 사용하는 이유는 무엇인가?
데이터베이스에서 prepared statement을 사용할 때 쓰이는데, 사용자가 입력한 데이터가 SQL 명령어로 인식되지않도록 막는 역할이 있기 때문이다.  prepare()에서 구조를 컴파일하고, execute()에서 데이터만 인식하여 실행하기 때문에, 동시에 실행되어 명령어 처리 되는 것을 막을 수 있다.
또한, 동일한 내용의 구조를 가진 데이터들을 매번 확인하고 실행하게 되면 그만큼 서버 자원이 소모되는 것이기 때문에 매번 쿼리의 구조를 분석하고 데이터를 확인할 필요없이 기존의 인식된 내용을 사용하여, 실행속도를 높인다. 이는 빨리빨리 민족의 한국인이 해당 페이지를 이용할 때, 빠른 로딩을 통해 막대한 만족감을 주는 것 중 하나이다. 

---

### 2. INSERT 와 UPDATE 의 차이점을 설명하시오.
두 명령어는 데이터와 관련되었다는 공통점이 있지만, 어떻게 관련되어있는지가 다르다.
INSERT의 경우, 테이블에 새로운 데이터를 추가할 때 쓰이므로, 조건이나 기존 데이터와 연관이 없지만.
UPDATE의 경우, 테이블에 존재하는 기존의 데이터를 수정할 때 쓰이므로, where를 통해 어떤 데이터를 어떻게 수정할지 미리 조건을 세우지않으면, 테이블에 존재하는 모든 데이터가 수정될 가능성이 존재하며, 있는 데이터를 수정하기 때문에 INSERT과 달리 기존의 데이터와 밀접한 관계를 가진다. 

---

### 3. 파일 업로드 시 MIME 타입, 확장자, 파일 크기를 검사해야 하는 이유는 무엇인가?
보안과 자원 보호를 위해 사용한다.
서버에 악성코드가 깔리거나 과한 용량의 파일이 올라가지않도록, 확장자(ex. .php, .png)의 종류를 제한하거나 파일의 크기를 제한하는 방식이 있다. 이는 악성코드를 방지하거나 서버 다운을 방지하는 역할을 맡는다.
또한, 사용자가 컴퓨터에서 확장자 이름만 바꿔서 서버에 올리고, 이를 다른 컴퓨터에서 다운받아서 월래 확장자로 바꾸는. 일명 택갈이를 피하기위해 파일의 실제 구조 확인을 통해 등록된 파일의 확장자가 파일 내부의 내용과 일치하는지 확인하는 방식이 있다.

---

### 4. session_start() 의 역할은 무엇인가?
세션을 시작하는 역할로, 세션을 시작하겠다고 서버에 알리는 역할을 한다.
이를 통해, 로그인 상태처럼 페이지의 위치에 상관없이 유지되야하는 사용자 관련 정보를 서버 공간에 저장하고, 추척할 수 있도록 세션 메커니즘이 활성화된다.
또한, 세션이 시작되면 서버에서 사용자를 식별하는 고유키(기본 값)인 세션 아이디를 생성해서 브라우저에 쿠키 형태로 전달한다. 이후 브라우저는 요청이 들어올 때마다, 해당 세션 아이디를 발송하여 누구인지 식별 가능하도록 한다.
